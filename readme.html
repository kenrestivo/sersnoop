sersnoop

what?

a simple command-line tool for linux that echoes bytes to and from any two serial ports, ptys, or network sockets, and prints all traffic to stdout, in hex and ascii.

why?

i needed a tool to redirect programs that only understand serial ports (or pty's) to communicate over the network instead (to a terminal server)

status

  • alpha. it works swimmingly well on linux.

    to install:

  • you'll need linux. the code uses non-portable linux-isms in the network socket code. this is easy to fix, but i haven't yet. for now, it's a penguin-only thing.
  • get the source and untar it.
  • get my silly utility header file and put it either in the source tree or somewhere else in your include path
  • type "make". hopefully it will build for you ;-)
  • "./sersnoop -h" will get you the help. you can specify two fd's to listen to: -a and -b. either or both of these can be a network socket, a physical tty, or a virtual pty.

    bugs

  • it only works on linux
  • the poll loop (and the program) exits when one fd goes away. i'd like it not to do that.
  • display code doesn't yet show the host/port of the socket(s)
